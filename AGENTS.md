# AGENTS.md — guide for AI coding agents working in this repo

## Stack

TypeScript + Node.js (ESM, run via `tsx`; typecheck via `tsc --noEmit`).
`verify_audit.py` (Python) is the provided grading gate — DO NOT MODIFY it.

## Commands

- Install: `npm install`
- Typecheck: `npm run typecheck` (or `npx tsc --noEmit`)
- Unit tests: `npm test` (node:test over `test/**/*.test.ts`)
- Full pipeline (offline): `make demo` (or `SEED_DIR=seed npx tsx src/pipeline.ts`)
- Verify gate: `make verify` (or `python3 verify_audit.py --audit out/audit.json --transcripts transcripts --schema audit.schema.json`)
- Probes: `make probe-approval|probe-agent-failure|probe-budget|probe-append-only|probe-idempotency`
- Trace/replay: `make trace ID=REC-001` / `make replay ID=REC-016`
- Eval: `make eval`

NOTE: `make` must be on PATH (it is in the Docker image / linux graders). On a
shell without `make`, invoke the underlying `npx tsx ...` commands directly (see Makefile).

## Layout

- `src/agents/` — orchestrator, worker, verifier, router, contracts (typed IO + roster)
- `src/stages/` — intake, orchestration, assembly, review, delivery
- `src/utils/` — hash (canon/sha MUST match verify_audit.py), amendment, stats, injection, schema, transcript, eventlog, clock, config
- `src/cli/` — trace, replay, eval, probe-* entry points
- `src/pipeline.ts` — the 5-stage runner (`runPipeline`)
- `test/unit/` — unit tests
- `out/`, `transcripts/` — runtime-generated (gitignored); regenerated by `make demo`

## Invariants (must never break)

- `canon`/`sha` in `src/utils/hash.ts` are byte-identical to `verify_audit.py`'s — verified by `test/unit`.
- Amendment is DERIVED from CASE_ID (`src/utils/amendment.ts`); never hardcode role/threshold.
- Outlier = robust IQR fence (k=3), never a hardcoded threshold.
- Nothing is delivered without a Verifier `pass` verdict + `approved` in the trail.
- Event log is append-only (`seq` strict 0..n-1); `out/` is cleared each run for idempotency.
- Never edit `seed/` or `verify_audit.py`.

## Gotchas

- The amendment for CEDX-33ACA8 is `risk_officer @ $44,000` (NOT finance_controller @ $18k).
- pdfjs-dist is used for PDFs (pdf-parse is broken on these files; was removed).
- Determinism depends on the monotonic clock (`src/utils/clock.ts`) — don't use `new Date()` in output paths; use `nowIso()`.
